# 変更要望 (2026-07-21): 実装ギャップの要求撤回と並列実行の要求反映

背景: AS-BUILT 照合で検出した実装ギャップ DIST-002 / DIST-003 (docs/todo.md) について、
プロダクトオーナーが「実装しない (要求を撤回する)」と決定した。あわせて v1.5.0 で
実装・リリース済みの組込み parallel プロセスタイプを要求・要件へ反映し、
arch の「逐次実行のみ」制約記述との矛盾を解消する。

## 1. ファイルログ要求の撤回 (DIST-002)

- 撤回対象: 「ログを .stfw/stfw.log へ集約」「日次ローテーション」「terminal 実行時の
  カラー出力 (ファイルはプレーン)」という要求 (USDM REQ-006 のログ仕様のうち該当部分)。
- 決定: 実装しない。stderr への構造化ログ (slog + secret マスキング) と、
  HTML レポート・OTLP トレースによる実行状況の可視化で運用要求を満たすと判断した。
- REQ-006 のうち実装済みの仕様 (ログ出力の存在・ログレベル・マスキング等) は維持する。

## 2. timezone 要求の撤回 (DIST-003)

- 撤回対象: stfw.timezone を業務日付判定・レポート時刻表記に使うという要求。
- 決定: 実装しない。時刻はプロセスのローカルタイムゾーンを正とする (現実装どおり)。
  stfw.yml の timezone キーは env 公開のみの任意キーとして残る (実装は参照しない)。

## 3. 並列実行の要求反映 (v1.5.0 で実装・リリース済み)

- 変更: 「v1.0 は逐次実行のみ (並列実行は将来の拡張余地)」という制約記述を改訂し、
  「実行モデルは逐次実行。ただし組込み parallel プロセスタイプにより、プロセスディレクトリ
  配下に定義した子プロセスの並走を選択できる」を要求として反映する。
- 実装済みの仕様 (正本: docs/AS-BUILT.md §4.14):
  - 子プロセスは親と同形式 `_{seq}_{group}_{type}`。子 1 件 = ジャーナル上の 1 ステップ
  - ステータス集約は Error > Warn > Success の悪い方 (worse-wins)。子同士に Blocked は無い
  - max_parallel (0 = 上限なし) を設定チェーンで解決 (stfw.yml 既定 → プロセス config 上書き)
  - 互換境界 (ディレクトリ規約 / プラグイン実行契約 / エビデンス規約 / ジャーナル) は不変
- 位置づけ: 単一実行 (1 run = 1 プロセス) 内の子プロセス並走であり、複数ユーザーの
  同時アクセスやスループットのスケールアウト (NFR B.x の対象) は引き続きスコープ外。
  NFR グレードの引き上げは意図しない。
